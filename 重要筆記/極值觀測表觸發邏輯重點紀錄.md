# 極值觀測表 (dgObserver) 觸發邏輯重點紀錄

這份極值觀測表（`dgObserver`）的運作邏輯，核心精神是在市場發生**「局部極端價格」**時，觀測其**「速度」與「成交量」**的變化，藉此抓出潛在的**「反轉點」**。

所有計算與狀態判定主要封裝在 `TradingEngine.CalcSimulationResultsInternal` 與 `TradingEngine.GetDurationsFull` 兩個方法中。以下為你不參考筆記，直接由程式碼分析得出的詳細計算邏輯：

### 1. A點的定義與過濾 (極點觀測)
系統會持續掃描 Tick 行情，尋找潛在的轉折點（我們稱為 A 點），分為兩種路徑：

*   **抓新高反轉 (做空)**：
    *   以當下 Tick 的價格為基準（A點）。
    *   **往前看 (Pre)**：蒐集 `N` 筆同盤別的**「外盤」**成交紀錄。如果往前找的過程中，有價格高於 A 點，代表 A 點不是局部最高點，立刻捨棄。
    *   **往後看 (Post)**：蒐集 `N` 筆對立盤別的**「內盤」**成交紀錄。如果往後找的過程中，價格又突破了 A 點，代表反轉失敗，立刻捨棄。
*   **抓新低反轉 (做多)**：
    *   邏輯完全對稱。往前找 `N` 筆**「內盤」**（不能有更低價），往後找 `N` 筆**「外盤」**（不能有更低價）。

### 2. 觸發條件：量增且速增
當成功收集滿前段 (`Pre`) 與後段 (`Post`) 各 `N` 筆 Tick 時（即程式碼中的 `actPre >= currentN && actPost >= currentN`），系統會計算並比對兩個關鍵數值來決定是否真正觸發並寫入觀測表：

1.  **平均速度 (`Avg`)**：
    *   將相鄰兩筆 Tick 的時間差加總後取平均。
    *   條件：`post.Value < pre.Value`。意思是**「後段的平均時間間隔，必須小於前段」**，也就是行情反方向的速度必須比原方向更快（速增）。
2.  **累積成交量 (`Vol`)**：
    *   將這 `N` 筆 Tick 的成交口數加總。
    *   條件：`preVol < postVol`。意思是**「反方向的成交量，必須大於原方向」**（量增）。

只要同時滿足 **「收集滿N筆未破價」** + **「反轉後速度變快」** + **「反轉後成交量放大」**，這筆紀錄就會正式誕生。

### 3. B點 (確認點) 與防守停損價 (StopLoss)
*   **觸發點 (B點/TrigPrice)**：當後段 (`Post`) 收集到第 `N` 筆 Tick 的那一瞬間，系統視為「確認達標」。這一筆 Tick 的價格與時間就會被記錄為 `TrigPrice` 與 `TrigTime`。
*   **預設防守價 (StopLossPrice)**：
    *   做空（抓新高）時，防守價預設綁定為 A 點的價格（`PrevHigh`）。
    *   做多（抓新低）時，防守價預設綁定為 A 點的價格（`PrevLow`）。

### 4. 階段三：時序停損與破價狀態機
觸發條件成立後，進入後續行情的監控（防守邏輯）：
系統會按時間順序，掃描 B 點之後的所有最新 Tick：
*   **做空模式**：如果後續有任何一筆 Tick 的價格 **大於 (`>`)** 當時的 A 點防守價。
*   **做多模式**：如果後續有任何一筆 Tick 的價格 **小於 (`<`)** 當時的 A 點防守價。

一旦發生上述情況，系統判定防守失敗，程式會將該筆觀測標記為 `IsBroken = true`（已破價），並記錄 `BreakTime`。
這時介面（`dgObserver`）的 `StopLossDisplay` 欄位，就會從原本的單純數字，變更為顯示 `{價位}(已破)`。
